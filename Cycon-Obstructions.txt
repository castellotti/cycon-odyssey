COG Engine Game Module - Obstruction File
-----------------------------------
Header Notes

Rules for dealing with Obstructions
 - Obstructions may exits in multiple Directional Objects at one time (including within the same room)


Obstruction # : int                          [Required]
Obstruction Name : String                    [Required]
Aliases : String                             [Optional]
Environment GraphicURL : String              [Either this or Textual Description is Required]
Environment Graphic Coordinates (X,Y) : int, int [Required if GraphicURL is provided]
CloseUp GraphicURL : string                  [Optional]
Text Description : String                    [See GraphicURL]
Type : String                                [Optional]
Locations : String                           [Required]
Visible : boolean                            [Required] (Required true in order to manipulate)
Notes : String                               [Optional]

Obstruction # : 5
Obstruction Name : Darkness
Text Description : You can't see in the dark, you need light!
Locations : Room(110)Direction[South]
Visible : No
Notes : This "obstruction" could have been implemented in the same manner as (for instance) the distance from the ledge over the river that runs through the mountains to the tree trunk on top of the mountain, but I felt
 that since the situation in which one needs a portable light source comes up so often, and is sovled by having
 a certain item as opposed to performing a different task, that implementing Darkness as an invisible obstacle
 my be better suited than setting up a series of Modifying ToWhichRoom Directional Objects. Implenting Darkness
 in this way was actually the orginal way in which all such problems where address, and I also wanted to show
 that there are many different ways to accomplish basically the same thing by mastering the Event Grammar
 of the COG Engine. By using this implementation, an ObstructedCompassGraphic will show that someone is capable
 of entering the darkness, but they currently cannot. If such a technique was used in the case of the tree
 stump example, the player would be given a hint that there must be *some* way to get up there.
